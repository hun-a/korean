# 초등학교 1학년에게 한글을 가르쳐보자

초등학교 1학년에게 **한글을 읽고 쓰게 만드는 가장 효과적인 방법**은 의외로 복잡하지 않습니다.
교육 연구와 실제 초등 교육에서 공통적으로 강조되는 핵심은 **“소리 → 글자 → 단어 → 문장” 순서 + 재미있는 반복 노출**입니다. 특히 한글은 **소리 기반 문자(phonetic writing system)**라서 원리를 이해하면 빠르게 읽을 수 있습니다. ([Discovering Mommyhood][1])

아래는 **가장 효과적인 실제 교육 방식 (부모가 집에서 할 수 있는 방법)**입니다.

---

# 1️⃣ “자음·모음 소리”부터 가르치기 (가장 중요)

처음부터 단어를 외우게 하면 어려워합니다.
**글자 = 소리**라는 것을 먼저 이해해야 합니다.

예시

* ㄱ → “그”
* ㄴ → “느”
* ㅏ → “아”

그리고 바로 결합을 보여줍니다.

* ㄱ + ㅏ = 가
* ㄴ + ㅏ = 나
* ㄷ + ㅏ = 다

👉 이걸 **“한글 조립”처럼 보여주면 효과적**입니다.

예

```
ㄱ + ㅏ = 가
ㄱ + ㅗ = 고
ㄱ + ㅜ = 구
```

✔ 한글은 이런 **조합 원리만 이해하면 읽기가 쉬운 문자**입니다. ([howtostudykorean.com][2])

---

# 2️⃣ “따라 쓰기”는 실제로 매우 효과적

연구에서도 **한글 초급 학습에서 “따라 쓰기(copying)”가 학습 효과가 높은 방법**으로 나타났습니다. ([ResearchGate][3])

추천 방식

1. 글자 보여주기
2. 읽어보기
3. 따라 쓰기
4. 다시 읽기

예

```
가 가 가
나 나 나
다 다 다
```

단,
⚠️ 너무 많이 쓰게 하면 싫어합니다.

추천

* 하루 10~15글자 정도

---

# 3️⃣ “아이 이름”으로 시작하면 효과가 매우 좋음

아이들은 **자기 이름에 가장 관심이 많습니다.**

예
아이 이름이 **민준**이라면

민
준

그리고

* 민 → 미 + ㄴ
* 준 → 주 + ㄴ

이렇게 확장합니다.

연구에서도 **아이 이름 글자를 활용한 한글 학습이 효과적**이라고 보고됩니다. ([Korea Science][4])

---

# 4️⃣ “단어 → 문장”은 짧게 시작

단어 예

엄마
아빠
나무
사과

문장 예

```
엄마가 와요
아빠가 가요
나무가 커요
```

이걸 **반복적으로 읽게 합니다.**

초기 읽기에서는
**같은 패턴 문장이 반복되는 텍스트**가 도움이 됩니다. ([위키백과][5])

---

# 5️⃣ 집에서 가장 효과적인 놀이 방식

아이들은 **공부보다 놀이에서 훨씬 빨리 배웁니다.**

추천 놀이

### 한글 찾기

집에서

* 우유
* 라면
* 과자

이 글자 읽기

### 한글 카드

카드 뒤집기

```
사 → 사과
나 → 나무
```

### 한글 벽 만들기

벽에 단어 붙이기

```
문
창문
의자
책
```

자주 보면 기억이 강화됩니다. ([위키백과][6])

---

# 6️⃣ 가장 중요한 원칙 (부모들이 많이 실수하는 것)

❌ 하루 1시간 공부
❌ 쓰기 3페이지

이건 거의 실패합니다.

대신

✔ 하루 10~15분

✔ 재미있게

이게 훨씬 빠르게 배웁니다.

# 프로젝트 설명
- 이 프로젝트는 초등학교 1학년이 한글을 체계적이고 재미있게 학습할 수 있도록 학습을 도와주는 웹 서비스 입니다.
- React로 개발된 static HTML로 구성되며
- 이 프로젝트는 Github pages 에 배포되어 서비스 됩니다.

## **필수 사항**
- 초등학교 1학년생이 좋아할만한 시각적 자료를 최대한 활용하세요.
- 너무 화려하지 않지만 너무 촌스럽지 않은 UI를 만드세요.
- 태블릿에서도 동작할 수 있도록 터치를 통한 사용자 interaction이 잘 되도록 구현하세요.
- 학습당 10분~15분 분량으로 동작할 수 있도록 만드세요.

## 현재 구현 기준 (다음 세션 필수 반영)

### 1) 배포/아키텍처
- 런타임 서버 없이 **정적 사이트**로 동작해야 함.
- GitHub Pages 대상 URL 기준: `https://hun-a.github.io/korean/`
- Vite는 `base: './'`로 설정되어 있어야 함.
- CI(GitHub Actions)는 **빌드 + 배포만** 수행하고 `generate:audio`를 실행하지 않음.

### 2) 데이터/콘텐츠 소스
- 수업 원본: `docs/lectures/001.md` ~ `030.md`
- 정적 JSON 생성:
  - 스크립트: `scripts/generateLessonData.mjs`
  - 출력: `client/public/lessons/*.json`
- 프론트는 `/api`가 아니라 **정적 JSON**을 fetch 해야 함.

### 3) 오디오(TTS) 운영 규칙
- 실시간 TTS 호출 금지. **사전 생성 mp3**만 사용.
- 오디오 생성 스크립트:
  - `scripts/generateLectureAudio.mjs`
  - 출력: `client/public/audio/*.mp3`, `client/public/audio/manifest.json`
- 고정 TTS 설정:
  - model: `gpt-4o-mini-tts`
  - voice: `coral`
  - instructions: 현재 스크립트에 정의된 고정 instruction 사용
- 기본 생성 범위는 MVP 기준 `1~3일차`이며, 필요 시 `--all`로 전체 생성.
- 오답 안내 문구 `다시 써보세요`는 manifest에 항상 포함되어야 함.

### 4) 학습 UI/플로우 규칙
- 수업은 **15분(5+5+5)** 고정:
  - 1단계: 자음/모음 읽기
  - 2단계: 단어 읽기
  - 3단계: 따라쓰기
- 탭 전환 시 **중복 패널 금지**:
  - 현재는 단일 패널(오른쪽 스타일)만 보여야 함.
- 3단계에서만 캔버스 따라쓰기 UI를 보여야 함.
- 1~2단계는 카드+듣기 중심 UI를 유지.

### 5) 따라쓰기 규칙
- 가이드 글자는 캔버스에 연하게 표시하되, **낮은 회차일수록 더 잘 보이게** 표시:
  - 초반(1~10) 가장 진하게
  - 중반(11~20) 중간
  - 후반(21~30) 더 연하게
- 오답 시:
  - 시각 피드백: 다시 써보기 문구
  - 청각 피드백: `다시 써보세요` mp3 재생(다른 재생 중이어도 강제 재생 가능)
- 정답 시:
  - 박수소리(`client/clap.mp3`) + 축하 효과

### 6) 3단계 완료 처리
- `다음 글자`는 무한 반복 금지.
- 각 회차에서 준비된 글자를 모두 소진하면 완료 UI 표시:
  - 메시지: `오늘 공부는 모두 완료했어요. 🥳`
  - 버튼: `다시 연습하기`, `1단계로 이동`

### 7) 개발 체크리스트
- 코드 수정 후 기본 확인:
  1. `npm run build`
  2. 로컬 preview에서 1~3단계 핵심 플로우 확인
  3. 오디오 manifest 경로/엔트리 누락 여부 확인
- 배포 장애가 나면 우선순위:
  1. `BASE_URL`/상대경로 처리
  2. `lessons/*.json` 및 `audio/manifest.json` 존재 여부
  3. manifest의 `availableDays`와 UI 회차 비활성화 상태 일치 여부

### 8) 1단계 입모양 자료 정책 (중요)
- 1단계의 입모양 자료는 **애니메이션이 아니라 정적 사진**으로 제공한다.
- 입모양은 벡터 도형/SVG로 합성하지 않고, 실사 스타일 이미지를 사용한다.
- 이미지 경로:
  - `client/public/mouth-photos/a.jpg`
  - `client/public/mouth-photos/eo.jpg`
  - `client/public/mouth-photos/o.jpg`
  - `client/public/mouth-photos/u.jpg`
  - `client/public/mouth-photos/eu.jpg`
  - `client/public/mouth-photos/i.jpg`
- 1단계 컴포넌트:
  - `client/src/components/MouthShapeAnimator.jsx`
  - 글자 선택 시 해당 정적 사진을 크게 보여주고, `소리 듣기` 버튼은 기존 TTS mp3 재생 로직을 그대로 사용.
- UI 문구:
  - 표시 문구는 `입모양 애니메이션`이 아니라 `발음 입모양 사진`을 사용.
  - 강의 활동 텍스트에 기존 문구가 남아있어도 렌더링 시 `입모양 사진`으로 치환해서 보여준다.
- 이미지 운영 원칙:
  - 웹 배포 성능을 위해 원본 PNG 대신 JPG를 사용한다.
  - 해상도는 1024x1024 유지, 과도한 용량은 압축해 관리한다.
  - 파일명/경로를 변경하면 `MouthShapeAnimator.jsx` 매핑도 반드시 함께 수정한다.
